## 핵심 설계 결정 요약 (다음 세션 참고용)

- Playwright CLI 벤치마킹: commander.js + 모듈별 명령어 등록 + 서비스 계층 분리
- HTTP POST `/mcp` 엔드포인트 주력 사용
- Accept 헤더에 `application/json, text/event-stream` 필수 (누락 시 406)
- `/mcp` 응답이 Streamable HTTP(Content-Type: text/event-stream)로 반환 → `event: message\ndata: ` 접두사 제거 후 JSON 파싱 (`/sse` 엔드포인트와 무관)
- 세션 ID를 `~/.figma-cli/session.json`에 캐싱
- nodeId 미지정 시 Figma Desktop에서 현재 선택된 노드 사용
- fileKey 파라미터 불필요 (로컬 MCP 서버 특성)
- 모든 명령어 `--json` 출력 지원
- 기본 타임아웃 60초 (get_design_context 대형 응답 대비)
- get_screenshot 응답은 content type이 image (다른 도구는 text)
- 에셋 URL (`localhost:3845/assets/*`)은 서버 실행 중에만 유효
