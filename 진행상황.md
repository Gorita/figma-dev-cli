## 핵심 설계 결정 요약 (다음 세션 참고용)

- SDK가 SSE 파싱, 세션 헤더 관리, initialize 시퀀스를 자동 처리 → 직접 구현 불필요
- `transportFactory` 옵션으로 테스트 시 mock Transport 주입
- Accept 헤더에 `application/json, text/event-stream` 필수 (누락 시 406)
- 세션 ID를 `~/.figma-cli/session.json`에 캐싱 (SessionManager)
- nodeId 미지정 시 Figma Desktop에서 현재 선택된 노드 사용
- fileKey 파라미터 불필요 (로컬 MCP 서버 특성)
- 기본 타임아웃 60초 (get_design_context 대형 응답 대비)
- get_screenshot: content type이 image → Buffer.from(data, 'base64')
- get_metadata: content가 2개 (XML + guidance)
- isError: true → ToolExecutionError throw
- MCP SDK `callTool` 반환이 union 타입 → `'content' in result` 가드 필요

## MCP SDK API 조사 결과

Explore 에이전트로 `node_modules/@modelcontextprotocol/sdk/` 타입 정의를 확인 완료:

### Client 클래스
```typescript
constructor(_clientInfo: Implementation, options?: ClientOptions)
// Implementation = { name: string; version: string }
connect(transport: Transport, options?: RequestOptions): Promise<void>
callTool(params: CallToolRequest['params'], resultSchema?, options?: RequestOptions): Promise<CallToolResult>
close(): Promise<void>
```

### StreamableHTTPClientTransport
```typescript
constructor(url: URL, opts?: StreamableHTTPClientTransportOptions)
// opts: { authProvider?, requestInit?: RequestInit, sessionId?: string, reconnectionOptions?, fetch? }
get sessionId(): string | undefined  // 연결 후 서버 제공 세션 ID 접근
```

### CallToolResult
```typescript
{ content: ContentBlock[]; isError?: boolean; structuredContent?: Record<string, unknown> }
// ContentBlock = TextContent | ImageContent | AudioContent | ResourceContent | ResourceLinkContent
// TextContent = { type: "text"; text: string }
// ImageContent = { type: "image"; data: string (base64); mimeType: string }
```
